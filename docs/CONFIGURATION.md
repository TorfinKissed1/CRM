# Переделка под другой бизнес

CRM спроектирована как переиспользуемое ядро. Чтобы адаптировать под другую сферу,
в большинстве случаев **код менять не нужно** — достаточно `config/crm.php` и Настроек.

## 1. Лейблы сущностей
`config/crm.php → labels`. Пример для барбершопа → фитнес-студии:
```php
'labels' => [
    'staff' => 'Тренеры',        // было «Мастера»
    'staff_singular' => 'Тренер',
    'services' => 'Абонементы',  // было «Услуги»
    'appointment' => 'Тренировка',
    // ...
],
```
Названия в навигации, заголовках и формах подхватятся автоматически (через `App\Support\Crm::label()`).

## 2. Включение/выключение модулей
`config/crm.php → modules`:
```php
'modules' => [
    'schedule' => true,   // false — скрыть «Расписание» из навигации и заблокировать маршрут
    'finance'  => true,
],
```

## 3. Тема и валюта (без правки кода)
Настройки → **Профиль бизнеса**: название, цвет темы, символ валюты. Хранится в таблице `settings`,
применяется в рантайме (цвет → CSS-переменная `--color-primary`).

## 4. Поля и логика
Если нужны доменно-специфичные поля — добавляются миграцией + в форму соответствующего модуля.
Бизнес-правила живут в `app/Actions` (например, `CompleteAppointment` создаёт операцию дохода).

## 5. Импорт своих данных
Настройки данных не требуют кода: **Клиенты → Импорт** принимает xlsx/csv с шагом сопоставления колонок
(дедуп по телефону). Колонки автоматически угадываются по заголовкам.
